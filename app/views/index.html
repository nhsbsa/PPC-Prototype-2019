Alternate start date pages

Validation options

Option 1 and Option 2 show what the start date page could look like if, after entering their personal details, we determined that the user had a current PPC that had not yet expired. Therefore the only dates they could pick would be future dates after the expiry of their current PPC.

Option 1 - A simple date entry as user would only be able to forward date their new PPC.

Option 2 - Two radio options giving the user the option to automatically start their new PPC to carry on from their current one, or to choose another date.

If we determined a user to not hold a current PPC they would be sent to the normal start date page (currently being that in Version 3).

No validation

Option 3 is a potential change to the start date page, which doesn't require identification of current PPC holders, but might reduce the number of overlaps.

Version 3 - Same as current iteration but with an added radio where users can select that they want their new certificate to carry on from their current certificate, and are prompted to enter the expiry date of their current certificate.

Their new start date would then be automatically set to the date after the expiry date of their current PPC.
